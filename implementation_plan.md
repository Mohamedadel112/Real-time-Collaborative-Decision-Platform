# Goal Description

The objective is to build a Real-Time Collaborative Decision Platform backend based on the provided [project_idea.md](file:///e:/Nestjs/Nestjs%20Projects/websocket%20projects/Collaborative%20Decision%20Platform/project_idea.md) PRD. The system requires real-time capabilities via WebSockets, a robust PostgreSQL database managed by Prisma, Redis for caching/rate-limiting/pub-sub, and various modules for Auth, Rooms, Decisions, Voting, Weight Calculation, and Reputation.

## Verification Plan

### Automated Tests
- Unit tests will be auto-generated by the NestJS CLI for each module and controller.
- Post-implementation, we will run `npm run test` to ensure modules instantiate correctly.

### Manual Verification
1. `docker-compose up -d` to start DB and Cache.
2. `npx prisma db push` to verify the schema syncing.
3. `npm run start:dev` to ensure the server starts seamlessly.
4. Using Postman or a WebSocket client (like Postman's WS feature or `wscat`) to connect and test connection, voting, and pub/sub.
